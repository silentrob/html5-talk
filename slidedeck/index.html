Animations

Tying everything together



Animations were first added in WebKit in 2007. In 2009,
the W3C updated the working draft and other browsers later added support.

Animation Browser Support

  • Safari 4
  • Chrome 4
  • FireFox 5
  • Opera 12
  • IE 10

Animation Syntax


animation: <name> <duration> <timing-function> <delay> <iteration-count> <direction>;
            

animation-name: myFirstAnimation;
animation-iteration-count: infinite; // 0 - n or infinite
animation-direction: alternate;
animation-play-state: running; // not well supported

animation-duration: 1s;
animation-timing-function: ease-in-out;
            

@keyframes myFirstAnimation {
  0% {
    padding: 0;
  }
  50% {
    padding: 0 20px;
  }
  100% {
    padding: 0 100px;
  }
}

NB: Vendor prefixes removed.

Alternate Animation Syntax


animation-name: mySecondAnimation;
animation-iteration-count: infinite; // 0 - n or infinite
animation-direction: normal;
animation-play-state: running; // not well supported

animation-duration: 1s;
animation-timing-function: ease-in-out;
            

@keyframes mySecondAnimation {
  from {
    padding: 0;
  }
  to {
    padding: 0 100px;
  }
}

NB: Vendor prefixes removed.

LocalStorage

API

  • setItem(key, val)
  • getItem(key)
  • removeItem(key)
  • length
  • key(index)
  • clear()

Availability/Security

  • domain
  • subdomain
  • port
  • protocol

Does not change scope

  • browser refreshes
  • url changes
  • query params

Use Cases

  • cache data for performance improvements
  • non-critical user preferences
  • offline support
  • non authenticated application data

Basic Example


localStorage.setItem("city", "Vancouver")
localStorage.getItem("city")
localStorage.removeItem("city")

Gotcha


var user = {
  name: "Brock Whitten",
  score: 12345
}

// don't do this
localStorage.setItem("user", user)
localStorage.getItem("user")

// do this
localStorage.setItem("user", JSON.stringify(user))
JSON.parse(localStorage.getItem("user"))
